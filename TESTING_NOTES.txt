PURPOSE
================================================================================

TEST ACCOUNT SETUP
================================================================================

The app is designed to allow new users to create accounts during the
onboarding flow. To test the app:

Option 1: Create a Test Account During Testing
- Open the app
- Tap "Create Account" on the login screen
- Enter test email: test@example.com (or any email)
- Enter password: TestPassword123!
- Complete the onboarding (6 steps)
- You'll have a working test account with sample data

Option 2: Use Pre-Populated Test Data
- The app includes sample groups and players
- If logging in with test@example.com, sample data is available
- This allows immediate feature testing without setup

================================================================================
KEY FEATURES TO TEST
================================================================================

1. AUTHENTICATION & ONBOARDING
   Steps:
   - Launch the app
   - Verify splash screen displays correctly
   - Tap "Create Account"
   - Complete 6-step onboarding:
     Step 1: Email entry
     Step 2: Password setup
     Step 3: Profile name
     Step 4: Preferred play style
     Step 5: Preferred days to play
     Step 6: Location (location access prompt)

   Expected Behavior:
   - Each step validates input
   - Navigation works between steps
   - Location prompt appears at step 6
   - Account creates successfully
   - User logged in automatically
   - Home screen displays

2. HOME SCREEN & GROUPS
   Steps:
   - View the home screen
   - See list of groups user belongs to
   - Verify group names and stats display
   - Tap a group to view details

   Expected Behavior:
   - Groups load correctly
   - Stats are accurate
   - No crashes on group selection
   - Group detail screen is responsive

3. MATCH CREATION
   Steps:
   - From group detail, tap "New Match"
   - Verify match creation form loads
   - Enter match details (opponent, score)
   - Tap "Save Match"

   Expected Behavior:
   - Match form displays all fields
   - Form validates required fields
   - Match saves and appears in history
   - Leaderboard updates immediately
   - No delays or performance issues

4. LIVE SCORING
   Steps:
   - Select a match in progress
   - Verify score display shows current points
   - Tap score buttons to increment/decrement
   - End the match

   Expected Behavior:
   - Scores update in real-time
   - All UI elements responsive
   - Score buttons are 44x44+ pixels
   - Match ends and is recorded
   - Stats update instantly

5. LEADERBOARD VIEWING
   Steps:
   - Open Leaderboard tab
   - View player standings
   - Sort by wins, losses, or win percentage
   - Filter by time period (if available)
   - Tap a player name to see details

   Expected Behavior:
   - Leaderboard loads quickly
   - Rankings are accurate
   - Sorting works correctly
   - Player detail page functional
   - No crashes on sorting/filtering

6. INVITES & PLAYER POOL
   Steps:
   - Open Invites tab
   - Browse available players
   - Verify player profiles display
   - Tap "Send Invite" on a player
   - Complete invite screen

   Expected Behavior:
   - Player list loads completely
   - Profiles display correctly
   - Invite form opens without errors
   - SMS credits shown (test environment shows free credits)
   - Note: SMS does not send in test environment (Twilio limitation)

7. LOCATION PERMISSIONS & BEACONS
   Steps:
   - Go to Settings
   - Verify Location Permission toggle
   - Grant location permission when prompted
   - View beacons feature (if implemented)
   - Verify nearby players display

   Expected Behavior:
   - Permission prompt appears once
   - Location access is requested correctly
   - Beacons feature functions (if available)
   - No location-related crashes

8. SETTINGS & PREFERENCES
   Steps:
   - Open Settings tab
   - Change profile information (optional)
   - Toggle notification preferences
   - Verify privacy policy link
   - Tap help/support links

   Expected Behavior:
   - Settings page loads
   - Changes save correctly
   - Help page links work
   - Privacy policy accessible
   - Links point to correct URLs

================================================================================
IMPORTANT: SANDBOX LIMITATIONS
================================================================================

** SMS MESSAGES DO NOT SEND IN SANDBOX **
- Twilio integration is configured to not send real SMS in test environment
- Invites can be created but won't deliver actual SMS messages
- This is intentional and Apple-approved testing limitation
- In production, SMS would be sent to actual phone numbers

** LOCATION SERVICES **
- Location requires either:
  a) Real iOS device with location enabled
  b) Simulator with mock location set
- Location won't work on simulator without mock location setup

ERROR HANDLING
================================================================================

The app includes error boundaries to catch and display errors gracefully:

- Network errors: Display user-friendly message
- Form validation: Shows specific validation errors
- Location errors: Gracefully handles denied permissions
- Server errors: Shows generic error message with retry option

If you see a red error screen, the app has encountered an unexpected error.
Note the error message and any stack trace for reporting.

================================================================================
PERFORMANCE EXPECTATIONS
================================================================================

- Screen transitions: < 500ms
- Leaderboard load: < 1 second
- Match save: < 500ms
- No noticeable lag during scrolling

DATA PERSISTENCE
================================================================================

The app uses local device storage and syncs with backend servers.

- Account data persists after logout/login
- Match history syncs across sessions
- Leaderboard data is server-authoritative
- Local cache updates when new data available

No data loss should occur during normal operation.
