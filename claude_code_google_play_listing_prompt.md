# Claude Code Prompt: Google Play Store Listing Copy

## Task
Write and deliver production-ready Google Play Store listing copy for **PlayPBNow** (bundle ID: `com.mcallpl.playpbnow`), a pickleball group management and real-time scoring app by Chipleball.

---

## Context

**What the app is:**
PlayPBNow is built for **recurring pickleball group organizers** — the people who run the Tuesday morning crew, the Friday evening round-robins, the private club sessions. It gives organizers a private group structure with a collaborative real-time multi-device scoring engine. No direct competitor has the scoring feature. Players are always free; organizers pay via an Organizer Pro subscription.

**Core differentiators:**
- Collaborative real-time multi-device scoring (unique in the market)
- Private recurring group structure (not a public court-finder)
- Longitudinal group data — tracks head-to-head records, win rates, rivalries over time
- iOS app is live on the App Store; Android just approved on Google Play

**Target buyer:** The organizer — the person who texts the group, sets the format, tracks the scores, and shows up every week. They are time-strapped and want less friction, not more features.

**Target player:** Permanently free. They join a group, show up, and play.

**Competitive landscape:** Pickleheads has VC backing and a USA Pickleball partnership and is building toward this space. PlayPBNow wins on the private-group + real-time scoring angle, not broad court discovery.

---

## Deliverables

Produce all four of the following in a single output file (`google_play_listing_copy.md`):

### 1. App Title (max 30 characters)
- Must include "pickleball" or a strong keyword
- Must communicate the core value
- Current bundle name is `com.mcallpl.playpbnow` — the display title needs to be set correctly

### 2. Short Description (max 80 characters)
- This is the preview text shown in search results before anyone taps the listing
- Must hook the organizer in one line
- Lead with the outcome, not the feature

### 3. Full Description (max 4,000 characters)
Structure it as follows:
- **Opening hook** (2–3 lines): State the problem organizers face. Make it visceral.
- **What PlayPBNow does** (1 short paragraph): Plain language, organizer POV
- **Key features** (bullet list, 5–7 items): Written as outcomes, not features. Bad: "Real-time scoring engine." Good: "Everyone sees the score update live — no more arguing about who won."
- **For players** (2–3 lines): Free forever, join any group you're invited to, see your stats grow over time
- **Social proof placeholder**: Leave a bracketed note like `[INSERT: first organizer testimonial]` so Chip knows where to drop real quotes as they come in
- **Closing CTA**: One crisp line driving organizers to start their free trial

### 4. Keyword Strategy (not a deliverable field in Play Store, but provide as reference)
- List 20 high-value search terms to work naturally into the description
- Flag which 5 are the highest priority
- Note: Google Play indexes the full description for keywords (unlike Apple's separate keyword field)

---

## Tone & Style Rules
- **Direct, confident, no filler.** Cut any sentence that doesn't earn its place.
- **Organizer-first.** Every paragraph should be readable by someone who runs a weekly group. The app saves them time and makes them look good.
- **No corporate speak.** No "seamless," "robust," "cutting-edge," or "intuitive."
- **Active voice throughout.**
- **Short sentences preferred.** This is an app store listing, not a whitepaper.

---

## Output Format
Save everything to: `google_play_listing_copy.md`

Include clear section headers. Under each deliverable, include:
- The copy itself
- A character count
- One line of rationale explaining the strategic choice made

Do not ask clarifying questions. Make the best decisions given the context above and deliver complete, paste-ready copy.
